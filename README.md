# SQL Trainer Platform

Учебная платформа для практического освоения SQL с возможностью загрузки тематических баз данных преподавателем, JWT-аутентификацией, визуализацией планов выполнения запросов и мониторингом активности студентов.

## 📋 Функциональность

### 🔐 Аутентификация и безопасность
- **JWT-аутентификация** с access и refresh токенами
- **Ролевая модель** (студент / преподаватель)
- **Фильтры авторизации** (JwtAuthFilter, TeacherFilter)
- **Хранение паролей** в зашифрованном виде (BCrypt)
- **JWT_SECRET** только через переменные окружения (без дефолтного значения)
- **Защита от XSS** (CSP заголовки, экранирование вывода, X-XSS-Protection)
- **Защита от SQL-инъекций** (валидация идентификаторов, quote_ident, regexp)
- **Защита от CSRF** (JWT в localStorage, браузер не подставляет автоматически)
- **Rate limiting** (ограничение частоты запросов для студентов, 1 запрос/сек)
- **Семафор** (ограничение параллельных запросов, 10 одновременных)
- **Таймауты запросов** (statement_timeout = 3 сек)
- **Ограничение строк** (MAX_ROWS = 1000)
- **Защита от опасных SQL команд** (блокировка DELETE, DROP, TRUNCATE, pg_sleep)
- **Валидация имён БД** (только буквы, цифры, подчёркивания)
- **Защищённые базы данных** (системные и демо-БД нельзя удалить)
- **Проверка доступа студентов к БД** (белый список разрешённых баз)
- **Изоляция соединений** (отдельные пулы для ADMIN, TEACHER, STUDENT)
- **Защитные HTTP-заголовки:**
  - Content-Security-Policy 
  - X-Content-Type-Options: nosniff 
  - X-Frame-Options: DENY 
  - X-XSS-Protection: 1; mode=block
- **Логирование подозрительных действий** (попытки доступа к чужим БД)

### 🧑‍🎓 Для студентов
- **Выбор базы данных** — переключение между учебными базами
- **Выполнение SELECT запросов** с ограничением по времени (3 сек)
- **Автодополнение SQL** (таблицы, колонки, ключевые слова)
- **Подсветка синтаксиса SQL** (CodeMirror)
- **Горячие клавиши** (`Ctrl+Enter` — выполнить, `Ctrl+Space` — автодополнение, `Ctrl+Shift+F` — формат)
- **Просмотр структуры базы** — список всех таблиц в выбранной БД
- **Кеширование результатов** (30 секунд, ускоряет повторные запросы)
- **Анализ плана выполнения** — визуализация `EXPLAIN ANALYZE`:
  - Древовидное представление плана с возможностью раскрытия/сворачивания узлов
  - Текстовый вывод PostgreSQL
  - Цветовое выделение типов узлов (Index Scan, Seq Scan, Join, Aggregate)
  - Отображение стоимости и времени выполнения
- **Экспорт результатов** — скачивание CSV с метаданными (время, количество строк, подпись)
- **Форматирование SQL** — автоматическое форматирование запросов
- **Подсказки при ошибках** — анализ ошибок и рекомендации по исправлению

### 👨‍🏫 Для преподавателя
- **Загрузка SQL-скриптов** для создания новых учебных баз данных:
  - Поддержка `.sql` файлов
  - Ограничение размера (2MB)
  - Защита от опасных SQL команд
  - Реальное время выполнения с **SSE-логами** (Server-Sent Events)
  - Прогресс-бар и детальный лог выполнения
- **Управление базами данных** — просмотр и удаление существующих БД
- **Защита от удаления** — системные и демо-базы (`sql_tutor_university_db`, `archaeology_10m`) нельзя удалить
- **Генерация студентов** — автоматическое создание учётных записей:
  - Генерация логинов (`student_1`, `student_2`, ...)
  - Генерация безопасных паролей (заглавные, строчные, цифры, спецсимволы)
  - Назначение группы
  - Экспорт сгенерированных данных в CSV
- **Управление студентами** — просмотр, редактирование и удаление
- **Мониторинг сессий** — отслеживание активных сессий студентов в реальном времени
- **Просмотр последних запросов** студентов
- **Личный профиль** — редактирование ФИО, email, смена пароля (доступен и студентам, и преподавателям)

## 🛠 Технологический стек

### Backend
- **Java 17** - основной язык
- **Apache Tomcat 9** - сервер приложений
- **Servlets / JSP** - обработка запросов и представления
- **JDBC** - доступ к данным
- **HikariCP** - пул соединений
- **JWT (jjwt)** - аутентификация
- **BCrypt** - хеширование паролей
- **Gson** - работа с JSON
- **SLF4J + Logback** - логирование

### Frontend
- **HTML5 / CSS3** - вёрстка и стили
- **Vanilla JavaScript** - клиентская логика
- **CodeMirror** - редактор SQL с подсветкой
- **Server-Sent Events (SSE)** - потоковые логи

### База данных
- **PostgreSQL 15** - основная СУБД
- **plpython3u** - генерация тестовых данных
- **pg_trgm** - полнотекстовый поиск

### Инфраструктура
- **Maven** - сборка проекта
- **Docker / Docker Compose** - контейнеризация
- **.env** - конфигурация через переменные окружения

### Тестирование
- **PowerShell / Bash скрипты** - нагрузочное тестирование
- **Тесты безопасности** (XSS, SQL injection, rate limiting)

## 🚀 Быстрый старт

### Требования
- Docker и Docker Compose
- (Опционально) Maven для ручной сборки

### Запуск

```bash
# Клонировать репозиторий
git clone <repository-url>
cd sql-tutor-platform

# Скопировать и настроить .env
cp .env.example .env
# Отредактировать .env (обязательно установить JWT_SECRET)

# Запустить развертывание
# Windows:
scripts\deploy\deploy.bat

# Linux:
./scripts/deploy/deploy.sh 
```
### После развертывания
1. Дождитесь завершения сборки (первый запуск может занять 30-40 минут из-за генерации тестовых данных)

2. Проверьте статус контейнеров:
```bash
cd docker
docker-compose ps
```
Оба контейнера (```sql_trainer_postgres``` и ```sql_trainer_app```) должны быть в статусе Up
3. Откройте приложение в браузере:
   * Главная страница: ```http://localhost:8081```
   * Страница входа: ```http://localhost:8081/login```

4. Войдите в систему:

|**Роль**	|**Логин**	|**Пароль**|
|-----------|-----------|----------|
|Преподаватель|	```teacher```	|```teacher123```|
|Студент	|```student_1```	|(генерируется при создании)|

5. Для преподавателя:
   * Загрузите SQL-скрипт для создания новой учебной базы 
   * Сгенерируйте студентов через панель управления 
   * Отслеживайте активные сессии студентов
6. Для студента:
   * Выберите учебную базу данных 
   * Напишите SQL-запрос (только ```SELECT```)
   * Нажмите ```Ctrl+Enter``` или кнопку "Выполнить"
   * При необходимости включите ```EXPLAIN``` для анализа плана запроса

### Переменные окружения
Все настройки задаются в файле ```.env ``` (скопируйте ```.env.example ``` и отредактируйте):
```bash
# База данных
DB_HOST=postgres
DB_PORT=5432
DB_ADMIN_PASSWORD=your_secure_password
DB_TEACHER_PASSWORD=your_teacher_password
DB_STUDENT_PASSWORD=your_student_password

# Безопасность (обязательно!)
JWT_SECRET=your_super_secret_key_min_32_chars

# Ограничения запросов
QUERY_TIMEOUT_SEC=3
MAX_ROWS=1000
MAX_CONCURRENT_QUERIES=10
```
Важно: Сгенерируйте уникальный ```JWT_SECRET``` командой:
```bash
openssl rand -base64 32
```
