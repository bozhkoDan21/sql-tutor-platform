# SQL Trainer - Учебная платформа

Учебная платформа для практического освоения SQL с возможностью загрузки тематических баз данных преподавателем, сессионной аутентификацией, визуализацией планов выполнения запросов и мониторингом активности студентов.

## 📋 Функциональность

### 🔐 Аутентификация и безопасность
- **Аутентификация преподавателя по паролю** — простая и безопасная, пароль задаётся через переменную окружения `TEACHER_PASSWORD`, без хранения в БД
- **HTTP-сессии** — управление состоянием через сессии браузера
- **Фильтры авторизации** (TeacherFilter для API, ViewResolver для JSP)
- **Хранение паролей баз данных** в зашифрованном виде (BCrypt)
- **Защита от XSS** (CSP заголовки, экранирование вывода, X-XSS-Protection)
- **Защита от SQL-инъекций** (валидация идентификаторов, quote_ident, regexp)
- **Rate limiting по сессии** (1 запрос/сек на сессию, защита от DoS)
- **Таймауты запросов** (statement_timeout = 3 секунды для студентов)
- **Ограничение строк** (настраивается преподавателем, по умолчанию 20)
- **Защита от опасных SQL команд** (блокировка DELETE, DROP, TRUNCATE, pg_sleep)
- **Лимит на количество учебных баз** (максимум 50 БД для предотвращения OOM)
- **Проверка доступа студентов к БД** (is_visible + период доступа + пароль)

### 🗂️ Управление базами данных (преподаватель)

- **Папки (категории)** — группировка баз по группам, типам заданий или семестрам. Поддерживается:
    - Создание папки
    - Редактирование названия
    - Удаление пустой папки
    - Перемещение баз между папками
- **Парольная защита** — каждая база может быть защищена паролем (BCrypt)
- **Видимость** — возможность скрыть базу от студентов (is_visible = false)
- **Период доступа** — задаётся дата начала и окончания доступа к базе
- **Отображаемое имя** — понятное название для студентов
- **Схема базы данных** — загрузка изображения со связями таблиц
- **Лимит строк для студентов** — max_rows (по умолчанию 20)

### 👨‍🏫 Для преподавателя

- **Полный SQL-доступ в тренажёре** — преподаватель может выполнять любые SQL-команды через страницу тренажёра
- **Загрузка SQL-скриптов** для создания новых учебных баз данных:
    - Поддержка `.sql` файлов
    - Ограничение размера (2 МБ)
    - Защита от опасных SQL команд
    - Реальное время выполнения с **SSE-логами** (Server-Sent Events)
    - Прогресс-бар и детальный лог выполнения
- **Загрузка схемы БД** — прикрепление изображения схемы для студентов
- **Управление папками** — создание, редактирование, удаление папок
- **Управление базами данных** — просмотр, редактирование метаданных и удаление существующих БД
- **Управление доступом** — установка пароля, видимости и периода доступа
- **Автоматическая генерация вопросов для Moodle** — загрузка текстового файла с вопросами и эталонными запросами, автоматическое выполнение запросов к выбранной БД, экспорт в форматы GIFT (рекомендуется), Moodle XML или текстовый просмотр

### 📚 Автоматическая генерация вопросов для Moodle

Функция позволяет преподавателю автоматически создавать тестовые задания для импорта в Moodle на основе эталонных SQL-запросов.

#### Формат входного файла

Входной файл должен быть текстовым (.txt) и содержать **пары строк**: строка с текстом вопроса, затем строка с SQL-запросом.

**Пример входного файла (`questions_example.txt`):**

```text
Найти всех студентов, родившихся после 2000 года
SELECT * FROM student WHERE birth_date > '2000-01-01' LIMIT 10;

Посчитать количество студентов в каждом городе
SELECT city_id, COUNT(*) as student_count FROM student GROUP BY city_id ORDER BY student_count DESC LIMIT 5;

Найти среднюю стипендию студентов, зачисленных в 2020 году
SELECT AVG(scholarship_amount) as avg_scholarship FROM enrollment WHERE year_of_enrollment = 2020 AND scholarship_amount IS NOT NULL;

Перевести лошадь "Атлант" в статус "на отдыхе"
UPDATE horses SET status_id = (SELECT id FROM horse_statuses WHERE name = 'на отдыхе') WHERE name = 'Атлант';
```
**Правила форматирования:**
- Каждая пара строк: текст вопроса → SQL запрос
- Пустые строки игнорируются 
- SQL-запрос может быть многострочным (но в файле рекомендуется писать в одну строку)
- Поддерживаются SELECT, UPDATE, INSERT, DELETE запросы

**Как это работает:**
1. Парсинг входного файла — система читает файл и извлекает пары "вопрос → запрос"
2. Выполнение эталонных запросов — каждый SQL-запрос выполняется к выбранной учебной базе данных
3. Вычисление хеша результата — для SELECT-запросов вычисляется SHA-256 хеш результата (усекается до 16 символов для удобства копирования)
4. Генерация выходного файла — создаётся файл в выбранном формате (GIFT, XML или текст)

**Почему хеш, а не полная таблица?**
При проверке заданий в Moodle сравнивается хеш результата, а не сама таблица. Это позволяет:
- Студенту не вводить громоздкие таблицы вручную
- Преподавателю быстро проверять правильность ответа
- При несовпадении хешей — экспортировать CSV и сравнить детально

**Пример сгенерированного GIFT-файла**

GIFT (General Import Format Technology) — это встроенный формат Moodle, работающий без дополнительных плагинов.

**Пример сгенерированного GIFT-файла (`moodle_questions.gift`):**

[/code]

**Структура GIFT-файла**

|**Элемент**	|**Описание**	|
|-----------|----------|
|```$CATEGORY: $course$/top/Название```|	Категория вопросов в Moodle	|
|```::Название вопроса::```	|Заголовок вопроса	|
|```[code lang="sql"]...[/code]```|Блок кода с SQL-запросом|
|```{ =правильный_ответ }```|Блок с правильным ответом|
|```~#```|Разделитель для комментария|
|Комментарий|Инструкция для преподавателя|

**Доступные форматы экспорта**

|**Формат**	|**Расширение**	|**Требования**	|**Рекомендация**	|
|-----------|----------|----------|----------|
|GIFT|	```.gift```	|Работает в любом Moodle|Рекомендуется|
|Moodle XML|```.xml```	|Требуется плагин CodeRunner	|Для продвинутых сценариев|
|Текстовый|```.txt```	|Только для предварительного просмотра|Для проверки преподавателем|

**Процесс использования**
1. Подготовка входного файла — создайте текстовый файл с парами "вопрос → SQL запрос"
2. Загрузка в SQL Trainer — выберите базу данных, укажите категорию, выберите файл 
3. Генерация — нажмите кнопку "Генерировать"
4. Импорт в Moodle — скачайте сгенерированный файл и импортируйте в Moodle через "Банк вопросов"
5. Проверка — студенты выполняют запросы в SQL Trainer и вводят хеши в Moodle

### 🧑‍🎓 Для студентов

- **Выбор папки и базы данных** — иерархическая навигация
- **Ввод пароля** — защищённые паролем базы требуют аутентификации
- **Просмотр схемы БД** — визуализация структуры базы (если загружена)
- **Просмотр списка таблиц** — все таблицы выбранной базы с возможностью вставки имени в редактор
- **Выполнение SELECT запросов** с ограничением по времени (3 сек) и максимальным количеством строк (настраивается преподавателем)
- **Ограничения безопасности** — студенты НЕ могут выполнять INSERT, UPDATE, DELETE, DROP и другие изменяющие данные команды
- **Автодополнение SQL** (таблицы, колонки, ключевые слова)
- **Подсветка синтаксиса SQL** (CodeMirror)
- **Горячие клавиши** (`Ctrl+Enter` — выполнить, `Ctrl+Space` — автодополнение, `Ctrl+Shift+F` — формат)
- **Просмотр структуры базы** — список всех таблиц в выбранной БД
- **Кеширование результатов** (30 секунд, ускоряет повторные запросы)
- **Анализ плана выполнения** — визуализация `EXPLAIN ANALYZE`:
    - Древовидное представление плана с возможностью раскрытия/сворачивания узлов
    - Текстовый вывод PostgreSQL
    - Цветовое выделение типов узлов (Index Scan, Seq Scan, Join, Aggregate)
    - Отображение стоимости и времени выполнения
- **Экспорт результатов** — скачивание CSV с метаданными (время, количество строк, подпись сервера SHA-256)
- **Форматирование SQL** — автоматическое форматирование запросов
- **Подсказки при ошибках** — анализ ошибок и рекомендации по исправлению

## 🛠 Технологический стек

### Backend
- **Java 17** — основной язык
- **Apache Tomcat 9** — сервер приложений
- **Servlets / JSP** — обработка запросов и представления
- **JDBC** — доступ к данным
- **HikariCP** — пул соединений
- **BCrypt** — хеширование паролей доступа к учебным базам данных
- **Gson** — работа с JSON
- **SLF4J + Logback** — логирование

### Frontend
- **HTML5 / CSS3** — вёрстка и стили
- **Vanilla JavaScript** — клиентская логика
- **CodeMirror** — редактор SQL с подсветкой
- **Server-Sent Events (SSE)** — потоковые логи при загрузке скриптов

### База данных
- **PostgreSQL 15** — основная СУБД
- **plpython3u** — генерация тестовых данных
- **pg_trgm** — полнотекстовый поиск

### Инфраструктура
- **Maven** — сборка проекта
- **Docker / Docker Compose** — контейнеризация
- **.env** — конфигурация через переменные окружения

### Тестирование
- **PowerShell / Bash скрипты** — тестирование безопасности и нагрузки
- **Тесты безопасности** (XSS, SQL injection, защита БД)

## 🚀 Быстрый старт

### Требования
- Docker и Docker Compose
- (Опционально) Maven для ручной сборки

### Запуск

```bash
# Клонировать репозиторий
git clone <repository-url>
cd sql-tutor-platform

# Скопировать и настроить .env
cp .env.example .env
# Отредактировать .env при необходимости

# Запустить развертывание
# Windows:
scripts\deploy\deploy.bat

# Linux:
./scripts/deploy/deploy.sh
```
### После развертывания
1. Дождитесь завершения сборки (обычно 1-2 минуты)
   *Примечание: если при развертывании вы выбрали загрузку тестовых данных,
   первый запуск может занять 30-40 минут (требуется ~2 ГБ дискового пространства)*

2. Проверьте статус контейнеров:
```bash
cd docker
docker-compose ps
```
Оба контейнера (```sql_trainer_postgres``` и ```sql_trainer_app```) должны быть в статусе Up
3. Откройте приложение в браузере:
   * Главная страница: ```http://localhost:8081```
   * Страница входа: ```http://localhost:8081/login```
   * Панель преподавателя: ```http://localhost:8081/teacher```

4. Войдите в систему:
   * Пароль: ```teacher123```

5. Для преподавателя:
   * Создайте папку для группировки баз
   * Загрузите SQL-скрипт для создания новой учебной базы
   * Настройте видимость, пароль и период доступа
   * Загрузите схему базы данных (изображение)
   * Отслеживайте активные сессии студентов
6. Для студента:
   * Выберите папку и базу данных
   * Введите пароль (если требуется)
   * Напишите SQL-запрос (только SELECT)
   * Нажмите ```Ctrl+Enter``` или кнопку "Выполнить"
   * При необходимости включите ```EXPLAIN``` для анализа плана запроса

### Переменные окружения
Все настройки задаются в файле ```.env ``` (скопируйте ```.env.example ``` и отредактируйте):
```bash
# База данных
DB_HOST=postgres
DB_PORT=5432
DB_ADMIN_USER=postgres
DB_ADMIN_PASSWORD=postgres
DB_TEACHER_USER=teacher_role
DB_TEACHER_PASSWORD=teacher_pass
DB_STUDENT_USER=students
DB_STUDENT_PASSWORD=student_pass

# Аутентификация преподавателя
TEACHER_PASSWORD=teacher123

# Ограничения запросов
QUERY_TIMEOUT_SEC=30
CONNECTION_TIMEOUT_MS=5000

# Ограничение параллельных запросов
MAX_CONCURRENT_QUERIES=10
SEMAPHORE_TIMEOUT_SEC=30

# Логирование
LOG_LEVEL=INFO
```
